# Tripo Help Center - Static Page Build Skill

## 目标
为 Tripo 中国版网站构建一个纯静态 Help Center FAQ 页面（单个 HTML 文件），无需后端服务，交给开发部署即可。

---

## 一、设计规范（Design Spec）

### 1.1 品牌素材

| 素材 | 文件/值 | 说明 |
|------|---------|------|
| Logo | `vast.png` | 横版白色文字+黄色T图标，用于页面顶部导航栏，深色背景 |
| Favicon | `1280X1280.PNG` | 正方形图标，用于浏览器标签页小图标 |
| 品牌主色 | `#F9CF00` | 按钮、链接、高亮、导航 active 状态、分类小标题 |
| 页面底色 | `#000000` | 全黑背景 |
| 卡片/区块背景 | `#1a1a1a` | FAQ 卡片、侧边栏 hover |
| 搜索框/表格背景 | `#242424` | 次级深色 |
| 边框色 | `#333333` | 卡片边框、分割线 |
| 主文字色 | `#ffffff` | 标题、问题文字 |
| 次文字色 | `#cccccc` | 答案正文 |
| 弱文字色 | `#999999` | 占位符、页脚、侧边栏未选中 |
| 支持邮箱 | `support@tripo3d.com` | 页脚联系方式 |

### 1.2 字体

| 语言 | 字体 | 回退 |
|------|------|------|
| 英文 | Noto Sans | sans-serif |
| 中文 | Noto Sans SC（思源黑体） | sans-serif |

加载方式：Google Fonts CDN
```html
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@400;500;700&family=Noto+Sans:wght@400;500;700&display=swap" rel="stylesheet">
```

### 1.4 交互规范

| 交互 | 行为 |
|------|------|
| 手风琴 | 点击问题展开/折叠答案，同一分类内可同时展开多个 |
| 搜索 | 纯前端 JS，实时过滤问题+答案文本，匹配项自动展开，无匹配显示空状态 |
| 侧边栏 | 点击平滑滚动到对应分类；滚动时自动高亮当前可见分类 |
| 插件卡片 | 悬浮上移 2px + 边框变主色，点击跳转飞书教程链接（新窗口） |
| 响应式 | 768px 以下隐藏侧边栏，单列布局 |

## 二、版本管理规范

### 2.1 版本号规则

```
vX.Y  X=大版本  Y=小版本

小版本（Y+1）：文案修改、样式微调、新增/删除单条 FAQ、素材替换
大版本（X+1）：整体结构重构、换配色方案、新增大功能模块、布局大改
```

### 2.2 Git 工作流

- 仓库地址：`https://github.com/cheryl0910/tripo-help-center`
- 分支：`main`（唯一分支）
- Git 用户：`cheryl0910` / `cheryl0910@users.noreply.github.com`

### 2.3 每次修改必须执行的步骤（Gate Checklist）

```
□ Step 1: 复述需求 — 告诉用户"这次改什么、不改什么"，等确认
□ Step 2: 修改代码 — 在 index.html 中执行修改
□ Step 3: 自检 — 确认修改正确、未破坏其他功能
□ Step 4: Git commit — 提交，commit message 格式：
          "vX.Y: 简要描述改了什么"
□ Step 5: Git push — 推送到 GitHub
□ Step 6: 变更摘要 — 告诉用户：改了什么、如何验证（刷新浏览器）
```

**严禁跳过任何步骤。Step 1 未获用户确认前不得进入 Step 2。**

### 2.4 回退流程

用户说"回退到 vX.Y"时：
1. `git log --oneline` 找到对应 commit
2. `git revert <commit>` 创建回退 commit（不用 reset，保留历史）
3. push 并告知用户

## 三、内容更新流程

### 3.1 FAQ 内容来源

- 飞书文档（通过 `lark-cli docs +fetch` 抓取）
- 用户直接粘贴的文本

### 3.2 内容变更类型

| 类型 | 操作 | 版本号 |
|------|------|--------|
| 修改某条 FAQ 文案 | 编辑对应 faq-item 的内容 | Y+1 |
| 新增一条 FAQ | 在对应分类末尾追加 faq-item | Y+1 |
| 删除一条 FAQ | 移除对应 faq-item | Y+1 |
| 新增一个分类 | 添加 section + 侧边栏导航项 | X+1 |
| 替换图片素材 | 替换文件 + 更新 HTML 引用 | Y+1 |
| 整体换风格/配色 | 修改 CSS 变量 | X+1 |

## 五、部署说明（给开发的）

1. 将 `index.html`、`vast.png`、`favicon.ico` 部署到服务器
2. 配置路由：`/help` 指向 `index.html`
3. 不需要任何后端服务、数据库或 API
4. 站内两处链接替换：
   - 插件使用 → `/help#api`
   - 资产迁移弹窗 → `/help#migration`
